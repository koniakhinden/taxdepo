# Учёт чеков → выгрузка для бухгалтера

Простой сайт с личным аккаунтом: фотографируешь чек → AI распознаёт **сумму, дату, категорию и магазин** → всё сохраняется в базу → одной кнопкой выгружаешь **Excel + архив всех фото** для бухгалтера (для налоговых списаний).

## Что внутри
- **Next.js 14** (React) — сайт и серверные API.
- **Supabase** — авторизация (email + пароль), база данных (PostgreSQL), хранилище фото. Есть Row Level Security: каждый пользователь видит **только свои** чеки.
- **OpenAI GPT-4o Vision** — автоматическое распознавание чека по фото.
- **Выгрузка** — `receipts.zip`: файл `receipts.xlsx` (таблица + итоговая сумма) и папка `photos/` со всеми чеками, привязанными к строкам.

---

## Шаг 1. Завести Supabase (бесплатно)

1. Зарегистрируйся на https://supabase.com → **New project**. Запиши пароль базы.
2. Когда проект создан, открой **Project Settings → API** и скопируй:
   - `Project URL` → это `NEXT_PUBLIC_SUPABASE_URL`
   - `anon public` key → это `NEXT_PUBLIC_SUPABASE_ANON_KEY`
3. Открой **SQL Editor → New query**, вставь всё из файла `supabase-schema.sql` и нажми **Run**. Это создаст таблицу чеков, политики доступа и хранилище фото `receipts`.

> Для быстрого старта можно отключить подтверждение email: **Authentication → Providers → Email → выключить "Confirm email"**. Тогда после регистрации сразу можно входить.

## Шаг 2. Завести ключ OpenAI

1. Открой https://platform.openai.com/api-keys → **Create new secret key**.
2. Скопируй ключ (`sk-...`) — это `OPENAI_API_KEY`. Пополни баланс на несколько долларов (распознавание одного чека стоит доли цента).

## Шаг 3. Запуск локально (для проверки)

Нужен установленный **Node.js 18+** (https://nodejs.org).

```bash
cd receipt-tracker
cp .env.local.example .env.local     # затем впиши свои 3 ключа в .env.local
npm install
npm run dev
```

Открой http://localhost:3000 → зарегистрируйся → добавь чек → выгрузи.

## Шаг 4. Деплой в интернет (Vercel, бесплатно)

1. Залей папку `receipt-tracker` в репозиторий на GitHub.
2. Зайди на https://vercel.com → **Add New → Project** → выбери репозиторий.
3. В разделе **Environment Variables** добавь три переменные из `.env.local`:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `OPENAI_API_KEY`
4. Нажми **Deploy**. Через минуту получишь ссылку вида `https://твой-проект.vercel.app` — открывается с телефона, можно фотографировать чеки прямо с камеры.

## Как пользоваться
- **Добавить чек** — сфоткать/выбрать фото. Поля заполнятся сами, можно поправить перед сохранением.
- **Список** — все чеки с превью, суммой и датой; удаление по крестику.
- **Выгрузить для бухгалтера** — скачивается `receipts_ГГГГ-ММ-ДД.zip` с таблицей и всеми фото.

## Настройка категорий
Список категорий — в `lib/categories.js`. Меняй под свой бизнес: AI будет выбирать значение из этого списка.

## Про безопасность
- Пароли и сессии — на стороне Supabase Auth (индустриальный стандарт).
- Приватные страницы закрыты через `middleware.js` — без входа не пустит.
- Row Level Security на уровне базы: даже зная ID чужого чека, получить его нельзя.
- Фото лежат в приватном bucket, доступны только владельцу по временным ссылкам.
- Секретные ключи (`OPENAI_API_KEY`) — только на сервере, в браузер не попадают.
